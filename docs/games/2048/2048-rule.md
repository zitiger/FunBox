## 2048 游戏规则

### 1. 游戏目标
在 4×4 的网格中，通过上下左右滑动合并相同数字的方块，最终制造出一个 **2048** 的方块。

### 2. 游戏板与方块
- **游戏板**：4行×4列共16个格子。
- **方块**：每个格子里可以有一张带有数字的方块，数字为2的幂（2, 4, 8, 16, …, 2048，甚至更高），或为空。
- 初始时，游戏板上会随机生成**两个方块**，数字为 **2** 或 **4**（以90%概率生成2，10%概率生成4）。

### 3. 移动与合并规则
玩家每回合可以选择向**上、下、左、右**四个方向之一滑动所有方块。

- **移动**：所有方块会尽可能向所选方向滑行，直到被网格边缘或另一个方块挡住。
- **合并**：如果两个**数字相同**的方块在滑行中相撞，它们会合并成一个新方块，其数字为两数之和。
  - 在一次移动中，每个方块最多只能参与一次合并。
  - 如果有三个相同数字方块连续相邻，靠滑动方向最近的两个会合并，第三个保持不动。
- 一次移动后，所有已合并的方块会锁定位置，该回合内不可再次合并。

**示例（向左滑动）：**
```
|2|2| |4|   →   |4|4| | |
合并过程：左边的两个2合并成4，同时原本的4也左移，但它们不再合并，最终留下两个4。
```

### 4. 新方块生成
每次玩家做出有效移动后，系统会在**空的格子**中随机生成一个新的方块：
- 90%概率生成数字 **2**；
- 10%概率生成数字 **4**。

如果移动没有导致任何方块位置变化（即没有空位且无法合并），则该移动**无效**，不会生成新方块。

### 5. 得分
- 每次合并产生的数字，会累加到当前分数中。
- 例如，合并两个8，得到16分。

### 6. 胜利条件
- 当任意一个方块上的数字达到 **2048**，即判定为**获胜**。
- 大部分版本允许玩家在获胜后**继续游戏**，挑战更高数字（如4096、8192等）。

### 7. 失败条件
- 当网格中**没有空格**，且**任意两个相邻方块（上下或左右）的数字都不相同**，无法进行任何有效移动时，游戏结束，判定为失败。

### 8. 其他规则（可选高级设计）
- **撤销**：可提供有限次数的撤销操作，回退到上一步。
- **最高分记录**：保存并显示历史最高分。
- **动画效果**：移动、合并、新生成方块时有平滑过渡动画，提升体验。
