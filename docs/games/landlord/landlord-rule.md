## 一、标准三人斗地主游戏规则

### 1. 用牌
一副完整的54张牌，包括大小王，**不剔除任何牌**。

### 2. 发牌与底牌
- 随机确定一名玩家先叫地主（可以轮换，第一局随机）。
- 每人发 **17张** 手牌，剩余 **3张** 作为底牌暂时扣下，待地主确定后翻开归地主所有。

### 3. 叫地主与抢地主
采用**叫分制**（简化而明确）：
- 系统随机选定首个叫牌者，按逆时针轮流叫分。
- 每位玩家只能选择：**“1分”、“2分”、“3分”或“不叫”**。
- 后叫者叫分必须**大于**前面已经叫出的最高分（若前面无人叫分，则可叫任意分或不叫）。
- 叫分最高者成为**地主**，底牌归其所有，手牌变为20张。其余两人为**农民**，手牌均为17张。
- 如果三家都“不叫”，本局作废，重新发牌。

### 4. 出牌规则
- 地主首出。
- 出牌必须为合法牌型，下家（按逆时针方向）可以选择：
  - 打出**相同牌型、相同张数、且牌点更大**的牌；
  - 或**不出**（过牌）。
- 当连续两家“不出”时，上一轮出牌者获得**新一轮自由出牌权**，可以打出任意合法牌型。
- 牌型定义（与通用规则完全一致）：

| 牌型 | 说明 |
|------|------|
| 单张 | 1张 |
| 对子 | 2张同点数 |
| 三条 | 3张同点数 |
| 三带一 | 3张同点数 + 1张单张（不能与三条点数相同） |
| 三带二 | 3张同点数 + 1对（对子点数不能与三条相同） |
| 顺子 | ≥5张连续单张，点数范围3~A（即最大顺10JQKA），不可包含2、小王、大王 |
| 连对 | ≥3对连续对子（如445566），不可包含2、小王、大王 |
| 飞机不带 | ≥2组连续三条（如333444），不可包含2、小王、大王 |
| 飞机带单 | 连续三条 + 同数量单张（单张不能是三条点数） |
| 飞机带对 | 连续三条 + 同数量对子（对子不能是三条点数） |
| 炸弹 | 4张相同点数 |
| 火箭 | 大王+小王（最大牌型） |

- **压制关系**：同类牌型必须用同类且更大点数压制；炸弹可以压制一切普通牌型；火箭最大，可以压制任何牌型包括炸弹。同类炸弹按点数比较大小。

### 5. 胜负与积分
- 任一玩家出完手牌，其所在阵营获胜：
  - 地主先出完 → 地主胜，两个农民负。
  - 任一农民先出完 → 农民阵营胜，地主负。
- 底分 = 叫分（1/2/3）。
- 倍率：每出现一个**炸弹**或**火箭**，总分倍率×2（可累积，上不封顶）。
- 最终得分 = 底分 × 倍率。地主胜则从两个农民处赢得相应分数；农民胜则两个农民各赢得相应分数，地主付出总分。

## 二、AI决策总框架

两个AI共享完全相同的决策算法，不进行作弊式手牌互通（农民AI之间也只能通过出牌逻辑隐式配合）。算法核心分为**手牌评估**、**叫地主**、**出牌引擎**、**农民配合**四个模块。

### 1. 手牌结构化评估
AI将手牌转化为标准内部表示，用于判断牌力。

**统计特征提取：**
- 点数分布：统计3,4,5,…,10,J,Q,K,A,2,小王,大王各有多少张。
- 特殊张数：王的数量、2的数量、A的数量。
- 牌型识别（不要求完整，只求“最大可能”）：
  - 炸弹（包括火箭）数量。
  - 最长的连续顺子长度（不考虑当前是否完整，按预分解思路）。
  - 可以组成的最大飞机带牌结构。
  - 散牌数量：无法组成任何牌型的孤立单张和孤对。

**牌力基础分（用于叫地主和出牌决策）：**
```
牌力 = 炸弹数×8 + 火箭×12 + 王的数量×5 + 2的数量×3 + A的数量×2 
      + 最长顺子候选长度×2 + 可组成飞机组数×3 - 散牌数×1.5
```
此分数可在0~80左右浮动，数值越大牌越好。

### 2. 叫地主策略
AI根据牌力分、牌型完整度与叫牌局势决定是否叫分及叫几分。

- **直接叫3分**（不犹豫，只要轮到自己就叫3分）：
  - 牌力分 > 22，且炸弹数 ≥ 2（含火箭）；
  - 或者手牌已存在“直接一手出完”的超级天牌倾向（比如长顺子+大飞机+双王，系统可做路径检测）。
- **叫2分或1分**：
  - 牌力分在12~22之间，且断点不多、补底牌后有显著增强可能。
  - 具体叫几分看牌力分高低和绝地反击意愿（可以设定1分、2分带随机性，避免太死板）。
- **不叫**：
  - 牌力分 < 12，炸弹少，王少，散牌太多，大概率无力当地主。

**动态调整（根据其他玩家叫分）：**
- 如果前面有人叫高分（比如2分或3分），说明对手牌力强，AI手中的牌力阈值需上调20%才可竞争。
- 如果自己是最后一个叫牌，且前面无人叫分，则只要牌力分 > 10 即可考虑叫1分“捡底”。

### 3. 出牌决策引擎（通用部分）
无论AI扮演地主还是农民，被动跟牌与主动出牌都遵循“高效走牌、控制权管理”的原则。AI会维护一个**牌型分解库**（启发式拆分，不强制求最优但有效率）。

#### 3.1 主动出牌（自己获得出牌权）
**决策流程：**
1. **一步获胜检测**：模拟全手牌能否通过一个或一连串合法牌型（如顺子+飞机带完）直接出完，如果能，直接打出第一手。
2. **处理零散牌（带牌）**：优先利用三带一、三带二、飞机带单/对，将孤立的单张和对子携带出去。  
   - 选择携带的牌时，保留更大的单张/对子以备后续控牌。
3. **打出连贯牌组（不用回收权）**：如果手中有一整条顺子或连对，且出完后剩下的牌易于控制，则优先出此类牌型，迅速减少手牌总数。
4. **试探牌权**：如果手牌中大牌多但零碎，可以先出一张**中等单张**（如10~Q），逼对手拆大牌。
5. **保守管理炸弹**：除非进入必胜路径，否则不轻易在第一手拆炸弹或主动出火箭（火箭常留到关键一手）。

#### 3.2 被动跟牌（响应上家出的牌）
**决策原则：**
- **能过则过**：如果手牌中有刚好大于上家的同型牌，且打出后对自己后续牌型影响不大，则跟出，拿回出牌权。
- **必须压制的局面**：
  - 如果上家（地主或农民）剩余牌数 ≤ 2，必须不惜代价（包括拆弹、拆王）压制，以防其直接获胜。
  - 如果自己是地主，上家农民出牌后手牌已很少（≤3张），必须用炸弹或大牌夺回出牌权改变牌型。
- **最小代价压制**：选择刚好大于上家且点数尽量低的牌（如人家出Q单，手上有K和A，优先出K）。保留更大的牌用于后续控场。
- **炸弹的使用**：
  - 当对手即将获胜时，炸弹用于打断出牌权；
  - 当自己即将用一系列牌直接赢下时，可用炸弹清场；
  - 其他情况下炸弹不宜轻易出手，尤其不能纯粹为了跟一手普通牌而使用炸弹。

#### 3.3 牌型选择内部模拟
AI在出牌前会对可能的几种合法出法进行简单模拟评分：
```
出法评分 = 手牌剩余总张数减少量 + 剩余牌型完整度保留值 + 牌权价值
```
选择评分最高的出法执行。（“牌权价值”指拿下出牌权后下一步能否连续出牌的增益）

### 4. 农民AI的专属配合逻辑
两个农民AI虽然有共同胜利目标，但**不能相互看到手牌**，只能从出牌过程中推断队友牌情。

**配合策略：**
- **顶牌**：当地主出小单张或小对子时，如果自己是地主下家，应尽量用较大的牌（但不必是最大）顶住地主，逼地主拆大牌，同时向队友传递“我有此类型控制力”的信号。
- **顺牌与送牌**：如果队友（AI判断另一个农民）获得出牌权并打出特定牌型，AI应尽量不压队友，除非确信自己能更快走完牌。
  - 判断依据：队友持续出同类型牌（如连续出单张），说明队友在该类型占优，AI应保留该类型大牌，让队友处理。
- **让牌**：当自己手牌较弱，而队友手牌不多时，即使可以压队友，也选择“不出”，让队友继续走牌。
- **关门炸**：当确认地主即将获胜时，无论谁手中有炸弹都必须炸掉，获得出牌权后立即打出队友可能接住的牌型（通常是单张或对子），帮助队友先走。

### 5. 记牌与推理模型（增强难度时启用）
AI会维护一个公开牌记录器：
- 记录已经打出的所有牌，动态计算剩余牌中可能存在的炸弹、2、王等关键牌。
- 例如，如果两个2已经打出，则AI知道剩下最多一张2，自己的A或K可用于控牌。
- 如果大小王都已出现，则火箭不存在，牌中最大变为2炸弹等。
- 根据对手的出牌历史推断其牌型结构（例如一直跟单张说明单张多），微调自己的出牌选择。

## 三、人类玩家与AI的交互流程（简要）
- 游戏开始，系统洗牌发牌，三名玩家（您+AI1+AI2）各17张，3张底牌留待叫地主。
- 叫地主阶段：系统随机指定起始叫牌者，若您是第一个，则提示您输入“1分/2分/3分/不叫”；若不是，您需等AI叫完后决定。界面显示三家叫分过程。
- 地主确定后，底牌公开，地主手牌变为20张。出牌权交地主。
- 每一轮出牌，轮到您时展示手牌，合法牌型按钮（或不出的选项）；轮到AI时，AI自动决策并展示出的牌。
- 直到一方牌出完，结算积分，进入下一局。
